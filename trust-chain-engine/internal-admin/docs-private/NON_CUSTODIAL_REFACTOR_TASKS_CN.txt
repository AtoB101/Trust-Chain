TrustChain 非托管结算改造任务拆分清单（中文）

目标：
- 从“池子托管模型”迁移到“钱包内锁仓模型”
- 资金不进入池子，确认后才 transferFrom

版本状态：执行态 v1.1（与当前代码同步）

适用：
- 创始人分发给早期贡献者
- 按优先级逐步推进

状态标签说明：
- [DONE] 已完成并已在仓库验证
- [PARTIAL] 已完成主要能力，仍有收尾
- [TODO] 待实现

==================================================
任务分层说明
==================================================
P0：必须先完成（核心安全与主流程）
P1：增强稳定性（可上线前必须做）
P2：体验与生态（上线后快速迭代）

每个任务建议字段：
- 任务ID
- 优先级
- 目标
- 验收标准
- 预估工时
- 依赖关系

==================================================
P0 核心任务（必须）
==================================================

[NC-001] 新增非托管结算核心合约骨架 [DONE]
优先级：P0
目标：
- 新建 NonCustodialSettlement 合约（或同等命名）
- 定义基础状态与事件
验收标准：
- 合约可编译
- 包含 bill 状态、reserved 映射、基础事件
预估工时：0.5-1天
依赖：无

[NC-002] 额度占用模型（reserved）实现 [DONE]
优先级：P0
目标：
- 实现 reserved[payer][token]
- 实现 available(payer, token) 计算
验收标准：
- 创建账单会增加 reserved
- available 计算通过单测
预估工时：1天
依赖：NC-001

[NC-003] createBill 非托管流程改造 [DONE]
优先级：P0
目标：
- createBill 不再转移资金
- 仅做额度占用 + 状态创建
验收标准：
- createBill 不调用 transferFrom
- bill.status=Pending
- 事件正确触发
预估工时：1天
依赖：NC-002

[NC-004] confirmBill 流程改造 [DONE]
优先级：P0
目标：
- 维持授权校验
- Pending -> Confirmed
验收标准：
- 未授权无法确认
- 状态迁移正确
预估工时：0.5天
依赖：NC-003

[NC-005] settleBill 执行转账实现 [DONE]
优先级：P0
目标：
- 仅在 Confirmed 时调用 transferFrom(payer, payee, amount)
- 成功后 reserved 减少 + 状态 Settled
验收标准：
- 正常结算成功并转账
- 重复结算失败
- 状态不可回退
预估工时：1-1.5天
依赖：NC-004

[NC-006] cancelBill 与 expireBill 释放占用 [DONE]
优先级：P0
目标：
- 支持取消与超时释放 reserved
验收标准：
- Cancelled/Expired 后 reserved 正确回退
- 超时边界测试通过
预估工时：1天
依赖：NC-003

[NC-007] 安全防护（重入/幂等） [DONE]
优先级：P0
目标：
- settle 路径防重入
- 状态机幂等保护
验收标准：
- 重入测试通过
- 重复调用不会导致资金或状态异常
预估工时：0.5-1天
依赖：NC-005

最终验收定义（非托管改造完成）
==================================================
满足以下条件即视为阶段完成：
1) 资金不进入合约池
2) createBill 仅占用额度
3) confirm 后才 transferFrom
4) cancel/expire 可释放占用
5) 关键安全测试通过
6) 前端和文档可支持外部开发者复现
