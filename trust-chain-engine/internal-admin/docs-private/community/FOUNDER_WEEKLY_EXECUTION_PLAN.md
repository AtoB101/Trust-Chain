# Founder Weekly Execution Plan (8 Weeks)

This plan is designed for a non-technical founder with clear vision but limited resources.

Goal: move TrustChain from solo MVP validation to a self-reinforcing builder community.

## Rules of Execution

1. Ship every week, no zero-progress week.
2. Public updates every week (even if progress is small).
3. Keep tasks small and verifiable.
4. Reward contribution quality, not noise.
5. Founder focus = direction, recruitment, transparency, consistency.

---

## Week 1 - MVP Proof and Public Baseline

### Objectives

- prove the MVP can run end-to-end
- make onboarding obvious for newcomers

### Founder Tasks

- run full deploy + smoke + UI flow once yourself
- confirm docs are consistent (`README`, deploy flow, community start page)
- publish "Project Baseline" post (what exists now)

### Deliverables

- updated repository state with working quickstart
- one public baseline announcement

### Success Criteria

- a new person can follow docs and reach "Health Check + Submit" on UI

---

## Week 2 - Open Task Board Launch

### Objectives

- convert project into contributor-ready task system

### Founder Tasks

- create 10 starter tasks (L1/L2) with acceptance criteria
- tag priority and estimated effort
- define response SLA for issues/PRs (e.g., <24h first response)

### Deliverables

- initial bounty/task board
- clear labels: `good-first-task`, `docs`, `frontend`, `tests`, `scripts`

### Success Criteria

- at least 3 tasks claimed by external contributors

---

## Week 3 - First External Contributions

### Objectives

- get first non-founder merged PRs

### Founder Tasks

- actively support first 3 contributors
- review PRs quickly and clearly
- publish "first contributors" recognition post

### Deliverables

- first external merged PRs
- contributor recognition section in weekly report

### Success Criteria

- at least 2 external merged PRs

---

## Week 4 - Governance and Trust Layer

### Objectives

- make decision-making and transparency predictable

### Founder Tasks

- open first public governance discussion thread (`TIP-001`)
- define decision cadence (weekly ops, monthly governance summary)
- publish first monthly transparency report using template

### Deliverables

- governance discussion in public
- report #1 published

### Success Criteria

- community can explain how decisions are made

---

## Week 5 - Improve Contributor Experience

### Objectives

- reduce friction for new builders

### Founder Tasks

- collect top 5 onboarding blockers from contributors
- fix docs/scripts for those blockers
- create "first PR in 60 minutes" mini path

### Deliverables

- onboarding improvements merged
- quick contributor path documented

### Success Criteria

- average onboarding time reduced

---

## Week 6 - Incentive System Activation

### Objectives

- make rewards consistent and transparent

### Founder Tasks

- run first bounty payout cycle (or points/reputation cycle if no funds)
- publish payout ledger (task -> reward -> recipient)
- announce next round with clearer criteria

### Deliverables

- transparent bounty ledger entry
- second-round bounty plan

### Success Criteria

- contributors trust reward process and keep participating

---

## Week 7 - Reliability and Community Confidence

### Objectives

- show protocol and process reliability under repeated execution

### Founder Tasks

- run and publish repeated deploy/smoke results
- publish incident/risk handling notes
- recruit 1-2 volunteer maintainers from active contributors

### Deliverables

- reliability update post
- maintainer candidate list

### Success Criteria

- community sees project is stable and maintained

---

## Week 8 - Transition to System-Driven Operations

### Objectives

- reduce founder bottleneck

### Founder Tasks

- delegate triage/review responsibilities
- lock weekly operating rhythm for next quarter
- publish "next 90 days" public roadmap update

### Deliverables

- shared ownership model
- next-quarter roadmap

### Success Criteria

- project keeps moving even when founder is less hands-on

---

## Weekly KPI Dashboard (Founder View)

Track these every week:

- active contributors
- external PRs opened/merged
- issue response time
- task completion rate
- deploy/smoke pass rate
- weekly settlement test success rate

## Weekly Founder Schedule (Low Resource Mode)

Minimum founder commitment:

- 30 min/day: issue and PR triage
- 2 sessions/week (60-90 min): contributor support + review
- 1 weekly report post (30-45 min)
- 1 roadmap/governance update per month

## Communication Templates

### Weekly Update (short)

- What shipped this week
- What is blocked
- Top 3 priorities next week
- Tasks open for contributors

### Contributor Recognition

- contributor name/handle
- contribution summary
- impact statement

## What to Avoid

- large unfocused tasks with unclear acceptance
- irregular communication
- hidden decision-making
- overpromising token/economic outcomes
- gating contributions behind private channels
