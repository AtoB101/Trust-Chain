# TrustChain Contributor Playbook

## Who This Is For

Developers, designers, writers, reviewers, and ecosystem contributors who want to help build TrustChain.

## How to Start

1. Read:
   - `README.md`
   - `docs/OPENCLOW_V01_DEPLOY_TEST_INSTRUCTIONS.txt`
   - `docs/community/VISION.md`
2. Pick one issue from the bounty/task board.
3. Comment "I want to take this task".
4. Submit PR with clear test notes.

## First Contribution Paths

- Fix docs clarity
- Improve deployment scripts
- Add tests for settlement edge cases
- Improve frontend console UX/monitoring

## PR Quality Standard

- keep change scope focused
- include test or reproducible verification steps
- explain why this change helps protocol/community
- avoid unrelated refactors

## Communication Standard

- be clear and respectful
- discuss trade-offs openly
- default to public channels for technical decisions
- document decisions after agreement

## Recognition

Contributors are recognized via:

- changelog credits
- community leaderboard
- bounty rewards
- governance reputation over time
