TrustChain 非托管模式测试网验收清单（CN）

目标：
- 在 Sepolia（或指定测试网）验证非托管闭环：清晰、安全、高效、可控。

==================================================
A. 部署与配置
==================================================
[ ] 1. 成功部署 `NonCustodialAgentPayment`
[ ] 2. 部署参数正确：
    - sellerBondBps = 3000（30%）
    - billTtlSeconds = 86400（24h）
    - arbitrator 地址正确
[ ] 3. 产物文件已生成：
    - `results/deploy-v01-eth.json`
    - `examples/v01-console-config.json`

==================================================
B. 非托管资金规则验收
==================================================
[ ] 1. `lockFunds` 后，用户钱包资金未转入协议池（仅状态变化）
[ ] 2. `active + reserved == locked` 全程成立
[ ] 3. 创建账单后：
    - 买方 reserved 增加 bill amount
    - 卖方 reserved 增加 30% bond
[ ] 4. `unlockFunds` 只能从 active 释放，不能动 reserved

==================================================
C. 账单生命周期验收
==================================================
[ ] 1. Pending -> Confirmed -> Settled 正常
[ ] 2. Pending -> Cancelled 释放双方 reserved
[ ] 3. Pending/Confirmed 超时后 -> Expired 并释放 reserved
[ ] 4. Confirmed -> Disputed -> ResolvedBuyer 正常
[ ] 5. Confirmed -> Disputed -> ResolvedSeller 正常
[ ] 6. Confirmed -> Disputed -> SplitResolved（首版必须）正常

==================================================
D. 安全与异常验收
==================================================
[ ] 1. 非法请求（无效 bill 状态）被拦截
[ ] 2. 触发 `InvalidTransferIntent` 事件并可在链上查询
[ ] 3. 重复结算尝试失败（无双花）
[ ] 4. allowance/balance 不足时结算失败并可定位原因
[ ] 5. 失败后账户状态仍满足不变式

==================================================
E. 前端与可观测性验收
==================================================
[ ] 1. 控制台可加载配置并显示 Non-custodial 地址
[ ] 2. 监测面板展示：
    - locked
    - active
    - reserved
    - invariant 状态
[ ] 3. 日志分类含 `invalid_transfer_intent`
[ ] 4. 导出 JSON 含非托管 KPI 字段

==================================================
F. 性能与并发验收
==================================================
[ ] 1. 单用户多账单并发下状态一致
[ ] 2. 多对手方并发（A/B/C/D/E/F）状态一致
[ ] 3. 关键路径响应在可接受范围（按测试网现实延迟）
