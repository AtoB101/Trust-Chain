G) Non-custodial batch operation (UI practical flow)
Open:
  http://localhost:8787/examples/v01-metamask-settlement.html
In the "Batch control (Non-custodial)" section:
  1) Read batch:
     - Input Batch ID
     - Click "Read batch" and confirm:
       - status = Open / Closed / Settled
       - totalPending and billCount are expected
  2) Close batch:
     - Click "Close batch"
     - Wait tx confirmed in Tx history
     - Read again and confirm status -> Closed
  3) Settle batch:
     - Set "Settle max bills" (0 means process all bills)
     - Click "Settle batch"
     - Read again and confirm:
       - status -> Settled (when all bills finalized)
       - totalPending -> 0 (when all eligible bills settled/finalized)
Acceptance quick checklist:
  - Close batch only works for batch owner wallet
  - Settle batch tx appears in Tx history
  - Logs include batch_state / batch_action entries
  - No invariant alarms in non-custodial KPI

8) Artifact Schema + Compatibility Notes
==================================================
To keep future runs stable and reproducible, use these artifact rules:

Required output files:
- results/run-050.json
- results/run-100.json
- results/aggregate-summary.json

Per-run artifact minimum fields (run-050/run-100):
- batchSize (number)
- successCount (number)
- failureCount (number)
- successRate (number, percentage or ratio, keep consistent)
- txHashes (string[])
- failuresByType (object map)
- startedAt (ISO timestamp)
- finishedAt (ISO timestamp)

Aggregate artifact minimum fields:
- totalRuns (number)
- totalTx (number)
- totalSuccess (number)
- totalFailure (number)
- aggregateSuccessRate (number)
- byRun (array of per-run summaries)
- generatedAt (ISO timestamp)

Compatibility rule:
- Standard field name is txHashes (NOT sampleTxHashes).
- If old artifacts exist with sampleTxHashes, either:
  1) migrate them to txHashes before aggregation, or
  2) delete and regenerate the run artifacts.

Operational defaults for stable stress runs:
- Use payer private key for settlement sender
- Recommended DELAY_MS=200 for large batch local/testnet runs
- Avoid mixing legacy artifacts with new parser versions
