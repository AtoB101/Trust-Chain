# Karma Bounty Framework v0.1

## Goal

Reward real, verifiable contributions that improve protocol quality, adoption, and reliability.

## Bounty Levels

### L1 - Starter

- scope: docs, small bug fixes, minor UX improvements
- expected effort: 2-8 hours
- reward: small fixed amount or points

### L2 - Builder

- scope: medium features, tests, tooling improvements
- expected effort: 1-3 days
- reward: medium fixed amount or points

### L3 - Core

- scope: critical protocol logic, security hardening, architecture-level upgrades
- expected effort: 3+ days
- reward: high amount (requires milestone review)

## Reward Components

- base reward (task completion)
- quality multiplier (tests, documentation, maintainability)
- impact bonus (adoption/reliability/security impact)

## Payment Rules

- reward only after merge and validation
- partial payouts allowed for milestone tasks
- rejected/incomplete tasks receive no payout

## Abuse Prevention

- no duplicate bounty payout for same issue
- no self-approved payouts
- suspicious behavior can trigger review freeze

## Public Ledger

Each payout should publish:

- contributor handle
- task ID
- reward amount
- tx hash (if on-chain payout)
- date and reviewer
